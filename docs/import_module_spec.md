# Import Module Specification

## Підтримувані формати
- Excel (.xlsx)
- CSV (.csv)
- YML / XML фіди

## Ціль
Зробити максимально гнучкий імпорт товарів, де користувач сам визначає:
- яка колонка або тег відповідає якому полю товару
- які поля треба заповнити примусово для всіх імпортованих товарів
- як обробляти відсутні значення
- що вважати оновленням існуючого товару

## Обов'язкові можливості
1. Попередній перегляд даних перед імпортом.
2. Вибір рядка заголовків.
3. Мапінг полів джерела на поля картки товару.
4. Можливість задати константу для будь-якого поля.
5. Пріоритет: константа / колонка / значення за замовчуванням.
6. Збереження шаблонів імпорту.
7. Режими імпорту:
   - створити нові товари
   - оновити існуючі
   - створити або оновити
8. Пошук дубля за:
   - артикулом
   - штрихкодом
   - назвою
   - зовнішнім кодом
9. Лог помилок імпорту.
10. Підтримка багатьох цін та додаткових атрибутів.

## Поля картки товару для мапінгу
- name
- sku
- barcode
- external_code
- category
- unit
- purchase_price
- retail_price
- wholesale_price
- internet_price
- currency
- vat_rate
- brand
- manufacturer
- country
- description
- warehouse
- quantity
- min_stock
- active

## Примусові значення
Користувач має мати змогу задати значення для всіх імпортованих товарів, якщо поле відсутнє в джерелі.
Приклади:
- currency = UAH
- unit = шт
- vat_rate = 20
- warehouse = Основний склад
- active = true

## Порядок заповнення поля
1. Якщо увімкнено `force_value`, брати його.
2. Інакше, якщо вибрана колонка/тег, брати з джерела.
3. Інакше, якщо є default_value, брати default_value.
4. Інакше залишити порожнім.

## Збережені шаблони
Шаблон імпорту має зберігати:
- тип джерела (excel/csv/yml)
- мапінг полів
- константи
- розділювач CSV
- лист Excel
- правило пошуку дубля
- режим імпорту

## Етапи UI майстра імпорту
1. Вибір файлу / джерела.
2. Попередній перегляд рядків.
3. Налаштування мапінгу.
4. Налаштування констант.
5. Перевірка результату.
6. Імпорт.
7. Звіт.
