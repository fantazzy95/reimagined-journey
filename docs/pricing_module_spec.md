# Pricing Module Specification

## Модуль: Встановлення цін

### Головний принцип
Модуль ціноутворення працює окремо від імпорту.
Імпорт оновлює закупівельні та службові дані.
Модуль `Встановлення цін` формує та записує продажні ціни в картку товару.

## Варіанти джерела ціни
Система повинна підтримувати гнучкий режим `C`:

1. **Рекомендована ціна постачальника**
   - якщо в прайсі постачальника є готова продажна ціна
   - можна брати її як основу

2. **Розрахунок від закупівельної ціни**
   - закупівельна ціна
   - плюс націнка
   - плюс за потреби округлення

3. **Змішаний режим / перемикач**
   - якщо є рекомендована ціна постачальника → використовувати її
   - якщо її немає → рахувати від закупівлі
   - або користувач вручну вибирає режим для документа

## Рівні правил націнки
Потрібно підтримати такі джерела націнки:
- по категорії
- по постачальнику
- по бренду
- ручна націнка в документі
- ручна ціна в документі

## Пріоритет правил
Рекомендований порядок:
1. ручна ціна
2. ручна націнка документа
3. правило по товару
4. правило по бренду
5. правило по категорії
6. правило по постачальнику
7. значення за замовчуванням

## Режими оновлення цін
- не змінювати ціни
- оновити тільки нові товари
- оновити тільки якщо змінилась закупівельна ціна
- оновити тільки якщо змінилась розрахункова продажна ціна
- оновити всі вибрані товари

## UI/UX документа
Користувач має бачити:
- поточну продажну ціну
- нову розраховану ціну
- джерело нової ціни
- правило націнки
- прапорець `застосувати`

## Результат проведення
Після проведення документа:
- в картку товару записується актуальна продажна ціна
- зберігається історія зміни цін
- документ лишається джерелом аудиту
