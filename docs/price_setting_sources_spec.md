# Price Setting Source Scenarios

## Головна ідея
Документ / майстер `Встановлення цін` повинен запускатись з різних джерел, а не тільки як окремий ручний документ.

## Підтримувані сценарії запуску

### 1. Ручний запуск
Користувач сам відкриває модуль `Встановлення цін` і:
- шукає товари широким пошуком
- фільтрує по категорії
- фільтрує по постачальнику
- фільтрує по бренду
- вручну відбирає товари
- застосовує правила ціноутворення

### 2. На підставі замовлення постачальнику
Користувач відкриває замовлення постачальнику і обирає:
- `Створити встановлення цін на підставі документа`

У майстер передаються:
- товари з документа
- закупівельні ціни
- валюти
- постачальник

### 3. На підставі надходження товару
Користувач відкриває документ надходження і обирає:
- `Створити встановлення цін на підставі надходження`

У майстер передаються:
- фактичні закупівельні ціни з приходу
- валюти
- список товарів

### 4. На підставі імпорту прайсу постачальника
Після імпорту прайсу користувач отримує вибір:
- тільки зберегти імпорт
- створити документ встановлення цін
- одразу перейти в майстер ціноутворення

У майстер передаються:
- імпортовані товари
- ціни з прайсу
- рекомендовані ціни постачальника (якщо є)
- валюти
- дані постачальника

## Вимоги до UI/UX
Майстер має показувати джерело запуску:
- ручний вибір
- замовлення постачальнику
- надходження
- імпорт прайсу

Також повинні бути доступні:
- масове виділення
- зняття виділення
- широкий пошук
- фільтрація
- попередній перегляд нової ціни
- можливість не застосовувати окремі рядки

## Архітектурний принцип
У всіх сценаріях повинен використовуватись один і той самий модуль розрахунку цін.
Змінюється лише джерело вхідних даних.

## Технічна модель
Потрібно передбачити поле:
- `source_type` (`manual`, `supplier_order`, `receipt`, `supplier_import`)
- `source_document_id` (nullable)

Це поле має зберігатись у документі `Встановлення цін` для аудиту.
